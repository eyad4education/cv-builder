# CV Builder

## Stack

- Client: Vite + React + React Router + Tailwind CSS + Capacitor (Android)
- Server: Express + MongoDB Atlas + Mongoose + JWT
- PDF: `jspdf` on the client

## Project Structure

```text
cv-builder/
|-- client/          ← Vite/React app + Capacitor Android project
|   |-- android/     ← native Android shell (generated by Capacitor)
|   |-- src/
|   |-- capacitor.config.ts
|-- server/          ← Express API
|   |-- railway.json ← Railway deployment config
|-- .github/
|   |-- workflows/
|       |-- build-apk.yml  ← GitHub Actions APK builder
|-- package.json
```

## Environment Setup

Create these files before running the app:

- `client/.env`
- `server/.env`

Use the provided `.env.example` files in each workspace as the template.

## Install

Run this once from the project root:

```bash
npm install
```

Because the repo uses npm workspaces, that installs dependencies for both `client` and `server`.

## Run in Development

From the project root:

```bash
npm run dev
```

Client default URL: `http://localhost:5173`

Server default URL: `http://localhost:5000`

## Production Build

```bash
npm run build
```

## Main Features

- Sign up and sign in with JWT authentication
- Protected dashboard editor
- Live CV preview
- Debounced autosave to MongoDB Atlas
- PDF download in the browser

## 🤖 Android APK

The Android app is built automatically via GitHub Actions on every push to `main`.

### Download the APK

1. Go to the **Actions** tab on GitHub
2. Click the latest **"Build Android APK"** workflow run
3. Scroll to **Artifacts** → download `cv-builder-debug-apk`
4. Install on Android (enable "Install from unknown sources" in settings)

### Before the first build — set these GitHub Secrets

Go to `Settings → Secrets and variables → Actions → New repository secret`:

| Secret | Value |
|---|---|
| `VITE_API_URL` | Your deployed Railway API URL, e.g. `https://cv-builder-api.up.railway.app/api` |

## ☁️ Deploy the Backend to Railway

1. Push this repo to GitHub (make it public)
2. Go to [railway.app](https://railway.app) → **New Project → Deploy from GitHub**
3. Select this repo → set the **Root Directory** to `server`
4. Add these environment variables in Railway dashboard:

| Variable | Value |
|---|---|
| `MONGO_URI` | `mongodb+srv://eyad4education_db_user:...@cluster0.we199yi.mongodb.net/` |
| `JWT_SECRET` | A long random string |
| `CLIENT_URL` | `*` |
| `PORT` | `5000` |

5. Railway will auto-deploy and give you a public URL like `https://cv-builder-api.up.railway.app`
6. Copy that URL into the `VITE_API_URL` GitHub Secret (append `/api`)

## 🌿 MongoDB Atlas — Allow All IPs (for dev)

1. Go to [cloud.mongodb.com](https://cloud.mongodb.com)
2. **Network Access → Add IP Address → Allow Access from Anywhere** (`0.0.0.0/0`)
3. This lets Railway (and local dev) connect without whitelisting specific IPs
